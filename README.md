# SoloDev Toolkit

혼자 개발할 때 반복되는 작업을 줄이고 개발 과정을 더 편리하게 만들기 위한 오픈소스 개발 보조 도구 모음입니다.

## 프로젝트 소개

SoloDev Toolkit은 개인 개발 과정에서 자주 발생하는 반복 작업과 번거로운 설정을 자동화하기 위해 시작한 프로젝트입니다.

프로젝트 생성, 파일 정리, 데이터 변환, 빌드 준비 등 개발자가 반복적으로 수행하는 작업을 간단한 도구로 만들어 관리하는 것을 목표로 합니다.

처음에는 개인적인 필요로 개발했지만, 비슷한 불편을 겪는 다른 개발자도 자유롭게 사용하고 개선할 수 있도록 오픈소스로 공개합니다.

## 개발 배경

혼자 프로젝트를 개발하다 보면 실제 기능 개발 외에도 다음과 같은 반복 작업에 많은 시간이 필요합니다.

* 프로젝트마다 반복되는 초기 설정
* 파일 및 폴더 정리
* 데이터 형식 변환
* 실행 파일 빌드 준비
* 로그와 결과 파일 확인
* 자주 사용하는 명령어의 반복 실행

SoloDev Toolkit은 이러한 작업을 자동화하여 개발자가 핵심 기능 구현에 더 집중할 수 있도록 돕는 것을 목적으로 합니다.

## 현재 제공 도구

### PDF 이미지 종합 도구

PDF 및 이미지 파일과 관련된 반복 작업을 하나의 프로그램에서 처리하기 위한 Windows용 통합 도구입니다.

Python 소스 코드와 PyInstaller 빌드 설정을 함께 제공하며, 일반 사용자는 GitHub Releases에서 단일 실행 파일을 다운로드하여 사용할 수 있습니다.

#### 주요 특징

* PDF 및 이미지 관련 작업을 하나의 프로그램에서 처리
* Windows용 그래픽 사용자 인터페이스 제공
* 단일 EXE 실행 파일 지원
* Python을 설치하지 않고 실행 가능
* 프로그램 소스 코드 공개
* PyInstaller 빌드 설정 파일 제공
* 인터넷 연결 없이 로컬 환경에서 실행 가능
* 사용자의 파일을 임의로 외부 서버에 전송하지 않음

> 세부 기능과 변경 사항은 각 버전의 릴리스 노트를 확인해 주세요.

#### 소스 파일 구성

```text
PDF_IMG_종합/
├─ PDF_IMG_종합.spec          # PyInstaller 빌드 설정
├─ SCAN.ico                  # 프로그램 아이콘
└─ Unified_Document_Tool.py  # 프로그램 소스 코드
```

다음 폴더는 프로그램 실행 및 빌드 과정에서 자동으로 생성되기 때문에 Git 저장소에 포함하지 않습니다.

```text
PDF_IMG_종합/
├─ __pycache__/  # Python 캐시
├─ build/        # PyInstaller 임시 빌드 파일
└─ dist/         # 완성된 실행 파일
```

완성된 EXE 파일은 Git 저장소가 아닌 GitHub Releases를 통해 배포합니다.

#### 실행 파일 다운로드

1. [GitHub Releases](https://github.com/qwwasdr0/SoloDevToolkit/releases) 페이지로 이동합니다.
2. 최신 버전의 Release를 선택합니다.
3. Assets 영역에서 `PDF_IMG_종합.exe`를 다운로드합니다.
4. 다운로드한 실행 파일을 더블 클릭합니다.

Windows에서 처음 실행하는 경우 Microsoft Defender SmartScreen 경고가 표시될 수 있습니다.

이 경우 파일의 출처와 릴리스 정보를 확인한 후 다음 순서로 실행할 수 있습니다.

1. `추가 정보` 선택
2. 게시자와 파일 이름 확인
3. `실행` 선택

#### 소스 코드로 실행

Python이 설치되어 있다면 저장소를 복제한 후 다음과 같이 실행할 수 있습니다.

```bash
git clone https://github.com/qwwasdr0/SoloDevToolkit.git
cd SoloDevToolkit/PDF_IMG_종합
python Unified_Document_Tool.py
```

사용하는 기능에 따라 별도의 Python 패키지 설치가 필요할 수 있습니다.

#### 직접 빌드

PyInstaller가 설치된 환경에서는 포함된 `.spec` 파일을 사용하여 Windows 실행 파일을 만들 수 있습니다.

```bash
cd PDF_IMG_종합
pyinstaller PDF_IMG_종합.spec
```

빌드가 완료되면 실행 파일은 다음 위치에 생성됩니다.

```text
PDF_IMG_종합/dist/
```

`build`와 `dist` 폴더는 자동 생성되는 빌드 결과물이므로 Git 커밋 대상에서 제외됩니다.

## 설치 방법

### 방법 1. 저장소 복제

Git이 설치되어 있다면 다음 명령어로 저장소를 복제합니다.

```bash
git clone https://github.com/qwwasdr0/SoloDevToolkit.git
cd SoloDevToolkit
```

### 방법 2. ZIP 파일 다운로드

1. GitHub 저장소 상단의 `Code` 버튼을 선택합니다.
2. `Download ZIP`을 선택합니다.
3. 다운로드한 ZIP 파일의 압축을 해제합니다.

ZIP 파일에는 소스 코드가 포함되며, 빌드된 EXE 파일은 포함되지 않을 수 있습니다.

### 방법 3. 실행 파일 다운로드

Windows 실행 파일이 제공되는 도구는 저장소의 [Releases](https://github.com/qwwasdr0/SoloDevToolkit/releases) 페이지에서 최신 버전을 다운로드할 수 있습니다.

## 사용 방법

사용하려는 도구의 폴더로 이동한 후 해당 도구의 실행 파일이나 스크립트를 실행합니다.

예시:

```bash
cd PDF_IMG_종합
python Unified_Document_Tool.py
```

Windows 실행 파일을 다운로드한 경우 다음 파일을 직접 실행합니다.

```text
PDF_IMG_종합.exe
```

각 도구의 상세 사용 방법은 해당 도구의 문서와 릴리스 노트를 확인해 주세요.

## 사용 예시

1. 필요한 도구를 선택합니다.
2. 처리할 파일 또는 프로젝트 경로를 지정합니다.
3. 작업 옵션을 설정합니다.
4. 작업을 실행합니다.
5. 결과 파일과 오류 로그를 확인합니다.

도구별 실제 사용 화면과 명령어 예시는 추후 `examples` 또는 `docs` 폴더에 추가할 예정입니다.

## 개발 원칙

이 프로젝트는 다음 원칙을 기준으로 개발하고 관리합니다.

* 반복 작업을 실질적으로 줄일 수 있는 기능을 우선합니다.
* 가능한 한 간단한 설치 및 실행 방법을 제공합니다.
* 인터넷 연결 없이 사용할 수 있는 기능은 오프라인 실행을 지원합니다.
* 오류 발생 시 원인을 확인할 수 있도록 로그를 제공합니다.
* 기능 변경 사항은 릴리스 노트에 기록합니다.
* 사용자의 데이터와 파일을 임의로 외부에 전송하지 않습니다.
* 소스 코드와 자동 생성되는 빌드 결과물을 구분하여 관리합니다.
* 실행 파일은 GitHub Releases를 통해 배포합니다.

## 기여 방법

버그 제보, 기능 제안, 문서 수정 및 코드 기여를 환영합니다.

1. 저장소를 Fork합니다.
2. 새로운 작업 브랜치를 생성합니다.

```bash
git checkout -b feature/기능명
```

3. 코드를 수정하고 테스트합니다.
4. 변경 사항을 Commit합니다.

```bash
git add .
git commit -m "feat: 기능 설명"
```

5. 자신의 저장소로 Push합니다.

```bash
git push origin feature/기능명
```

6. 원본 저장소에 Pull Request를 생성합니다.

자세한 기여 규칙은 `CONTRIBUTING.md`가 추가된 이후 해당 문서를 확인해 주세요.
